# Creating a free, Markdown-based, personal portfolio


## Why create a portfolio ? 

Goals:
- A place to show my projects (Useful for potential recruiters)
- A place to document my projects (Useful for me to consult later and for people with similar projects in mind)
  -  Selfhosted projects
  -  GameDev project
  -  Any random idea I want to try
-  If coded by oneself, also counts as a project too

## Building the portfolio

What we need:
- Your code editor of choice (I use VS Code)
- [Hugo](https://gohugo.io) framework
- A suitable theme (I picked [Hugo Coder](https://github.com/luizdepra/hugo-coder))
- A Github account (counts both as our repository and deploy server, using [Github Pages](https://pages.github.com))

(...)

Usual workflow:
- Open "write" workspace (markdown files only)
- Write on the intended blog file
- Commit & Push
- (auto) Github Action triggers, copying files to respective folders + deploy to Repo's Github Page
